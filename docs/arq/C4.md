# C4 - Detalhamento (Codigo)

## Estrutura de Arquivos
- backend/index.js
  - Bootstrap do servidor e carregamento do .env.
- backend/server.js
  - Servidor HTTP, integracao com SDK da brapi.dev, rotas REST (API apenas).
- backend/api/brapi.js
  - Integracao com SDK da brapi.dev.
- frontend/index.html
  - Layout da pagina, estrutura das tabelas e markup do painel de score.
- frontend/app.js
  - Logica de UI, cache local, chamadas API, painel de score e renderizacao.
- frontend/config.js
  - URL base da API usada pelo frontend.
- frontend/analysis/utils.js
  - Funcoes utilitarias para series de preco.
- frontend/analysis/trend.js
  - Calculo de tendencia.
- frontend/analysis/patterns.js
  - Identificacao de padroes graficos.
- frontend/analysis/volume.js
  - Confirmacoes por volume e nivel de preco.
- frontend/analysis/momentum.js
  - Calculo de momento do preco.
- frontend/analysis/risk.js
  - Calculo de risco/retorno.
- frontend/analysis/score.js
  - Score por confluencia de fatores.
- frontend/styles.css
  - Estilos e layout.
- frontend/state.js
  - Estado compartilhado do frontend.
- frontend/data/
  - Cache local e montagem de analises (inclui nome e logo do ativo).
- frontend/ui/
  - Renderizacao de UI e utilitarios (abas, resumo com logos, tabela com ordenacao).

## Endpoints
- GET /api/tickers
  - Retorna lista de tickers (filtrada por tipo).
- GET /api/quote?symbols=...
  - Retorna cotacoes para ate 10 tickers.
- GET /api/history?symbol=...&range=...&interval=...
  - Retorna historico de preco para um ticker.

> Todas as rotas acima tambem aceitam o prefixo `/api/v1`.
